Dubbing and Audio description Profiles of TTML2

This specification defines DAPT, a TTML-based file format for the exchange of timed text content in dubbing and audio description workflows.

Scope

This specification defines a text-based profile of the Timed Text Markup Language version 2.0 [[TTML2]] based on [[ttml-imsc1.1]]. This profile is intended to support dubbing and audio description workflows worldwide.

Introduction

Creating a dub is a complex, multi-step process that involves:

  • Transcribing and timing the dialogue in the original language from a completed show to create a source transcription text
  • Notating dialogue events with character information and other annotations
  • Generating localization notes to guide further adaptation
  • Translating the dialogue to a target language
  • Adapting the translation to the dubbing and subtitling specifications; ex. matching the actorâ€™s lip movements in the case of dubs and considering reading speeds and shot changes.

Creating audio description content is also a complex process with similar steps.

This specification defines DAPT, a TTML-based format for the exchange of timed text content among authoring and prompting tools in the localization and audio description pipeline. A DAPT file is designed to carry pertinent information for dubbing or audio description such as type of script, dialogues, descriptions, timing, metadata, original language text, transcribed text, language information, and to be extensible for future annotations. This specification first defines the data model (see [[[#data-model]]]) for DAPT scripts and then its representation as a TTML document with restrictions (see [[[#ttml-format]]]).

DAPT Data Model

This section specifies the data model for DAPT.

DAPT Script

A DAPT Script is a document resulting from an authoring workflow with the following properties:

Property Name Description Applicability (Dubbing Scripts, Audio Description Scripts, All) Mandatory/Optional
Script Type
  • Original Language Dialogue List: (Dubbing Script) Literal transcription of the dialogues and on-screen text in their original spoken/written language(s).
    If a movie contains dialogues in English and Hebrew, the Original Language Dialogue List will contain some Events in English and some in Hebrew. None of the events are translated.
  • Translated Dialogue List: (Dubbing Script) Translation of the dialogues or on-screen text in a common language. It is meant to be adapted to produce a Pre-Recording Dub Script. It may contain Events from the Original Language Dialogue List for context, to assist in the adaption process.
    If a movie contains dialogues in English and Hebrew, the French Translated Dialogue List will contain at least the translation in French of all Events. It may still retain Events in Hebrew and English to assist further processing.
  • Pre-recording Dub Script: (Dubbing Script) Result of the adaptation of a Translated Dialogue List for better lip-sync. It may also contain Events from the Original Language Dialogue List for context, to assist further processing.
  • As-recorded Dub Script: (Dubbing Script) Represents a transcription of the output of the dubbing workflow. It may also contain Events from the Original Language Dialogue List for context and quality verifications.

 All Mandatory
Primary Language Default language for the Text content of Events.

When multiple languages are used in a script, it can correspond to the language of the majority of events, to the language being spoken for the longest duration, or to the language arbitrarily chosen by the author.

 All Mandatory
Events List of timed text events corresponding to dialogues, on screen text, or descriptions. All Mandatory
Characters List of characters referenced in the events Dubbing Scripts Mandatory (for Dubbing Scripts)

It does not have to be unique for this Event. It may be used to identify in a human-readable way one or more Events that need to be processed together, e.g. in a batch recording.

TTML Format

This section defines the TTML format based on the data model defined in [[[#data-model]]].

Identification and Profile Signaling

TTML documents representing DAPT Scripts shall be compliant with the Text Profile defined in [[!ttml-imsc1.1]] with the additional constraints defined in this specification.

TTML documents representing DAPT Scripts shall specify a ttp:contentProfiles attribute on the tt element with one value equal to http://www.w3.org/ns/ttml/profile/dapt. Other values such as the text profile designator [[!ttml-imsc1.1]] of may be present.

DAPT Namespace

This specification defines the following namespaces:

Name Prefix Value
DAPT Metadata daptm http://www.w3.org/ns/ttml/profile/dapt#metadata

The namespaces defined by this specification are mutable [[namespaceState]]; all undefined names in these namespaces are reserved for future standardization by the W3C.

The namespace prefix values defined above are for convenience and Document Instances MAY use any prefix value that conforms to [[!xml-names]].

Other namespaces may be present, as needed and permitted by XML and TTML, use for either specific TTML attributes or elements, or for elements and attributes from other standards or proprietary languages.

Implementations shall ignore elements and attributes in the DAPT namespaces that they do not recognize.

Implementations shall ignore elements and attributes in non TTML-namespaces that they do not recognize.

Proprietary Metadata

Many dubbing and audio description workflows may need to annotate events or documents with proprietary metadata. This SHOULD be done, as permitted by [[!ttml-imsc1.1]], using foreign elements and attributes, in vendor-specific namespaces.

Document structure

A DAPT script object is represented by a TTML document with the following constraints:

  • the xml:lang attribute shall be present on the tt element and its value shall not be empty. It describes the primary language of the script.
  • The head element shall be present and a metadata element shall be present in the head element to provide some DAPT Script properties as follows:
    • A ttm:role attribute shall be present with one of the following values representing the Script Type:
      • x-DUBBING_ORIGINAL_DIALOGUE_LIST for Original Language Dialogue List
      • x-DUBBING_TRANSLATED_DIALOGUE_LIST for Translated Dialogue List
      • x-DUBBING_PRE_RECORDING for Pre-recording Dub Script
      • x-DUBBING_AS_RECORDED for As-recorded Dub Script

    <tt xmlns="http://www.w3.org/ns/ttml" 
    xmlns:ttp="http://www.w3.org/ns/ttml#parameter" 
    xmlns:tts="http://www.w3.org/ns/ttml#styling" 
    xmlns:ttm="http://www.w3.org/ns/ttml#metadata" 
    xmlns:dapt="http://www.w3.org/ns/ttml/profile/dapt" 
    xml:lang="en" 
    ttp:contentProfiles="http://www.w3.org/ns/ttml/profile/dapt">
  <head>
    <metadata ttm:role="x-DUBBING_DIALOGUE_LIST"/>
  </head>
  ...
</tt>

    For each Character, the following constraints apply:

    • There shall be a ttm:agent element, child of the metadata element, with the following constraints:
      • The type attribute shall be set to character.
      • The id attribute shall be present and set to the Character Identifier.
      • It shall contain a ttm:name element with its type attribute set to alias and its content set to the Character Name.
      • If the Character has a Talent Name property, the following applies:
        • Another ttm:agent shall be added to the metadata element, with the following:
          • its type attribute shall be set to person
          • its xml:id attribute shall be set. Its value SHOULD be derived from the Character Identifier
          • it shall have a ttm:name child element whose type shall be set to full and its content set to the Talent Name
        • the first ttm:agent shall have a ttm:actor child, with its agent attribute set to the xml:id of the first ttm:agent.
        <medadata>
  <ttm:agent type="character" xml:id="character_b89ea6a2-bcfa-46b4-a835-fc13081672e7">
    <ttm:name type="alias">BOOKER</ttm:name>
    <ttm:actor agent="actor_b89ea6a2-bcfa-46b4-a835-fc13081672e7"/>
  </ttm:agent>
  <ttm:agent type="person" xml:id="actor_b89ea6a2-bcfa-46b4-a835-fc13081672e7">
    <ttm:name type="full">Matthias Schoenaerts</ttm:name>
  </ttm:agent>
  <ttm:agent type="character" xml:id="character_dc56a9b2-81d6-4415-8d87-ec06450c5313">
    <ttm:name type="alias">DESK CLERK</ttm:name>
  </ttm:agent>
</medadata>
    • There may be a style element in the styling element providing the styles applicable to Events from this Character. The xml:id shall be present and represents the Character Identifier. Any style attribute from [[ttml-imsc1.1]] may be present. A ttm:agent attribute may be present to link the ttm:agent element representing the Character. 
      <styling>
  <style xml:id="style_0085df18-5020-4473-a312-fe4b233bad65"
         ttm:agent="character_0085df18-5020-4473-a312-fe4b233bad65"
         tts:color="#FFFFFF" tts:backgroundColor="#8F42AD"/>         
</styling>

    For each Event, the following constraints apply. There shall be one div element, with the following constraints:

    • The xml:id attribute shall be present containing the Event Identifier.
    • The style attribute may be present. If present, it may contain a reference to the style defining the Character Style. Additional style references or inline styles may be used.
    • The begin and end attributes shall be present and correspond to the Start and End of the Event.
    • The ttm:agent attribute may be present and if present, shall contain a reference to the ttm:agent representing the Character.
    • It may contain a metadata element with the following constraints:
      • The following attribute corresponding to the On Screen Event property may be present: 
        daptm:onScreen
  : "ON" 
  | "OFF"
  | "ON_OFF"
  | "OFF_ON"

      • A ttm:desc element may be present representing the Event Description.
      • Event Type

    • It shall contain one p element whose text content is the Text of the Event and shall have an xml:lang attribute corresponding to the Event Text Language. The text content may use TTML markup to represent the Text Style.
    • It may contain one p element whose text content is the Contextual Text of the Event. If present, it shall have an xml:lang attribute corresponding to the Event Contextual Text Language. It shall also have a ttm:role attribute whose value is x-context. The text content may use TTML markup to represent the Contextual Text Style.
      • The text content of the paragraph may contain TTML elements such as span or TTML attributes such as tts:ruby used to alter the layout or styling of each paragraph
    <div xml:id="event_0130df43-5443-4206-b1d9-d331e42b5a86"
     begin="9663f" end="9682f" 
     style="style_bb89e9c6-4453-4402-a376-5a496cfb9839"
     ttm:agent="character_bb89e9c6-4453-4402-a376-5a496cfb9839"
  <metadata daptm:onScreen="ON">
    <desc>Scene 1</desc>
  </metadata>
  <p xml:lang="pt-BR" >Você vai ter.</p>
  <p xml:lang="fr" ttm:role="x-context">Bah, il arrive.</p>
</div>

Layout

This specification does not put additional constraints on the layout and rendering features defined in [[!ttml-imsc1.1]].

Layout of the paragraphs may rely on the default TTML region (i.e. if no layout is used in the head element) or may be explicit by the use of the region attribute, if a region element is defined in a layout element in the head element.
